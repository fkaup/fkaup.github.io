Searching the 30c3 from Android using YaCy

30 Dec 2013

YaCy

YaCy is a free, decentralized P2P search engine, which allows circumventing censorship and intransparent ranking of search results. YaCy is fully configurabe, allowing indexing of manually chosen pages or using YaCy as a proxy to automatically index pages loaded through the proxy. The generated search results are stored in a DHT and such, also exchanged between peers in the YaCy network.

At the 30C3 a YaCy instance is deployed to search the event pages, schedule and the wiki from within the network. To support mobile users, an Android application, 30c3 Conference Search, is written.

API

The It makes use of the Apache Solr API, returning search results as well defined JSON objects. These objects are parsed within the Conference Search by the JSON parser provided by the Android SDK (i.e. org.json.*).

Android App

The 30c3 Conference Search is a simple search application. After startup, the application displays a search entry field above the is designed in the 30C3 color scheme to fit the general 30C3 appearance.

The search results are fetched from the YaCy instance co-located at the conference venue and displayed in a list view. The retrieved search results are formatted, the page title, description and search result nuber are displayed in a list view. The search results are opened in the default browser when clicked.

The 30c3 Conference Search also supports pagination. Only results which are displayed are fetched from the server, reducing the used bandwidth of the mobile device.

The source code of the 30c3 Conference Search is available on GitHub. It should easily be changed to support any Solr compliant servers or fetching search results from your personal YaCy instance.
